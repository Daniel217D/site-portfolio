# Demo API Specification

## Статус

Draft v0.1

---

# Назначение

Демо каждого проекта взаимодействует с отдельным внешним API.

Под каждый проект допускается полностью уникальная реализация demo-интерфейса.

---

# Домен API

Базовый домен API:

```text
demo.<site-domain>
```

Конкретный production-домен пока не определён.

---

# Конфигурация окружения

Обязательные переменные:

```env
NUXT_PUBLIC_DEMO_API_URL=
NUXT_PUBLIC_DEMO_API_KEY=
```

Правила:

* `NUXT_PUBLIC_DEMO_API_URL` доступен на клиенте.
* `NUXT_PUBLIC_DEMO_API_KEY` является публичным и доступен на клиенте по текущему замыслу.
* Production Node.js runtime не предполагается, поэтому demo-интеграция проектируется как клиентская.

---

# Дальнейшая детализация

OpenAPI-спецификация API будет добавлена позже.

После её появления необходимо создать отдельную спецификацию взаимодействия с API с описанием:

* контрактов запросов и ответов;
* обработки ошибок;
* таймаутов и ретраев;
* ограничений по безопасности и публичному доступу.
