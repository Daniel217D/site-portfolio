# Архитектурный подход

Используется FSD.

Проект организуется внутри каталога `src/`.

Структура верхнего уровня:

```text
src/
├── app/
├── pages/
├── widgets/
├── features/
├── entities/
└── shared/
```

Предварительная структура:

```text
src/
├── app/
│
├── pages/
│   ├── home/
│   └── project/
│
├── widgets/
│   ├── project-page/
│   ├── project-presentation/
│   └── project-demo/
│
├── features/
│   └── demo-api-request/
│
├── entities/
│   └── project/
│
└── shared/
    ├── api/
    ├── config/
    ├── lib/
    └── ui/
```

---

# Правила организации

* Архитектура Nuxt адаптируется под FSD-подход, а не наоборот.
* Страницы и UI-композиция строятся через слои `pages`, `widgets`, `features`, `entities`, `shared`.
* Общие инфраструктурные зависимости размещаются в `shared`.
* Модель проекта инкапсулируется в `entities/project`.
* Интеграция с demo API выносится в `features/demo-api-request` и `shared/api`.

# Проектные страницы

Каждый проект имеет:

* общий базовый шаблон;
* индивидуальную реализацию содержимого.

Полный набор секций пока не определён.

Предполагаемые секции:

* презентация;
* демо;
* дополнительные материалы.

Простое текстовое описание проекта размещается непосредственно в шаблоне страницы.

Презентация реализуется отдельным встраиваемым компонентом внутри страницы проекта.
