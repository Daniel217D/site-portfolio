# CI/CD Specification

## Статус

Draft v0.1

---

# Docker

Проект должен содержать Docker-конфигурацию для разработки.

Минимально:

```text
Dockerfile.dev
docker-compose.yml
```

Docker должен поддерживать:

* запуск dev-сервера;
* проверку типов;
* линтинг;
* unit-тесты;
* генерацию статической сборки.

---

# GitHub Actions

Публикация будет выполняться через GitHub Actions.

На текущем этапе:

* репозиторий отсутствует;
* production-сервер отсутствует;
* настройки публикации не определены.

Необходимо подготовить базовую структуру:

```text
.github/
└── workflows/
    ├── check.yml
    └── deploy.yml
```

`deploy.yml` на текущем этапе является шаблоном-заглушкой без реального деплоя.

---

# Требования к pipeline

`check.yml` должен быть ориентирован на запуск базовых проектных проверок:

* typecheck;
* lint;
* unit tests;
* static generate.

Конкретная реализация workflow определяется после появления репозитория.
