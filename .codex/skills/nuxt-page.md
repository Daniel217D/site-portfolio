# Nuxt Page

## Назначение

Этот skill описывает, как создавать новые страницы Nuxt в рамках текущего проекта.

Цель - делать страницы предсказуемыми по структуре и совместимыми с FSD и SSG.

## Маршруты проекта

Поддерживаются маршруты:

```text
/                главная страница
/p               каталог проектов
/p/:project_name страница проекта
```

`project_name` является основным идентификатором проекта в URL.

## Правила страницы

* Страница должна решать route-level задачу, а не хранить всю логику в одном файле.
* Простое текстовое описание проекта допускается размещать прямо в шаблоне страницы.
* Презентация подключается как отдельный встраиваемый компонент.
* Demo подключается как отдельный UI-блок или widget/feature-композиция.
* Страница проекта должна искать проект по `project_name`.

## Что держать в странице

Подходит для страницы:

* сборка layout и основных секций;
* связывание route params с доменными данными;
* подключение виджетов проекта;
* page-level SEO и метаданные.

Не подходит для страницы:

* низкоуровневые API-запросы;
* переиспользуемые UI-элементы;
* сложная интерактивная логика demo;
* инфраструктурные утилиты.

## SSG-ограничения

* Решение должно быть совместимо со статической генерацией.
* Нельзя полагаться на обязательный production Node.js runtime.
* Клиентская интерактивность должна деградировать предсказуемо, если внешний API недоступен.

## Чеклист новой страницы

1. Обновить спецификацию, если меняется маршрут или поведение страницы.
2. Выбрать место страницы в `src/pages`.
3. Подключить данные проекта через `project_name`.
4. Вынести презентацию и demo в отдельные компоненты, если они не тривиальны.
5. Проверить, что структура страницы не нарушает FSD-границы.
