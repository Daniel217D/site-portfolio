# Reveal Presentation

## Назначение

Этот skill описывает правила интеграции Reveal.js-презентаций в страницы проектов.

Цель - сделать презентации единообразными по роли в интерфейсе и не выносить их в отдельный режим без необходимости.

## Роль презентации

* Презентация является частью страницы проекта.
* Презентация встраивается как отдельный компонент.
* Презентация не должна требовать отдельного маршрута по умолчанию.

## Допустимое содержимое

Reveal.js-презентации могут содержать:

* анимации;
* пошаговое раскрытие контента;
* интерактивные элементы;
* графики;
* визуализации.

## Архитектурные правила

* Компонент презентации размещается в `widgets/project-presentation` или рядом с проектной страницей, если реализация узкоспецифична.
* Page-level код только подключает презентацию, а не хранит её внутреннюю логику целиком, если она заметно разрастается.
* Инициализация Reveal.js должна быть совместима с клиентским рендерингом и не ломать SSG.

## Практические рекомендации

* Инициализировать Reveal.js только там, где доступен браузерный DOM.
* Следить, чтобы без JS страница не становилась полностью бесполезной.
* Отделять контент слайдов от служебной логики инициализации там, где это помогает читаемости.
* Не смешивать demo API-логику с презентационным компонентом без необходимости.

## Когда обновлять спецификации

Обновляй спецификации, если:

* появляется отдельный режим показа презентации;
* меняется способ встраивания в страницу проекта;
* появляются обязательные общие правила для всех презентаций.
