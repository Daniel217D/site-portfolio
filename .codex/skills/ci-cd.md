# CI/CD

## Назначение

Этот skill описывает минимальные правила для Docker и CI/CD-инфраструктуры проекта.

Цель - не усложнять pipeline раньше времени, но сразу держать структуру совместимой с дальнейшей автоматизацией.

## Docker

Минимально проект должен содержать:

```text
Dockerfile.dev
docker-compose.yml
```

Docker-конфигурация должна поддерживать:

* запуск dev-сервера;
* проверку типов;
* линтинг;
* unit-тесты;
* генерацию статической сборки.

## GitHub Actions

Ожидаемая структура:

```text
.github/
└── workflows/
    ├── check.yml
    └── deploy.yml
```

Правила:

* `check.yml` запускает базовые проектные проверки;
* `deploy.yml` на текущем этапе является шаблоном-заглушкой;
* реальный production deploy пока не настраивается.

## Требования к `check.yml`

Workflow должен быть ориентирован на запуск:

* `npm run typecheck`
* `npm run lint`
* `npm run test:unit`
* `npm run generate`

## Практические ограничения

* Не проектировать CI вокруг обязательного production Node.js runtime.
* Не добавлять преждевременную сложность в deploy workflow до появления реальной инфраструктуры.
* Если меняется набор обязательных проверок, сначала обновить спецификацию `specs/testing.md` и при необходимости `specs/ci-cd.md`.

## Когда использовать skill

Используй этот skill, когда задача касается:

* Docker-конфигурации для разработки;
* GitHub Actions;
* автоматизации проверок;
* подготовки шаблонного deploy pipeline;
* согласования локальных команд и CI-процесса.
