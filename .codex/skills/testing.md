# Testing

## Назначение

Этот skill фиксирует минимальные требования к проверкам и качеству изменений.

Цель - помогать агенту завершать задачи с проверяемым результатом, а не только с написанным кодом.

## Обязательные команды

```bash
npm run typecheck
npm run lint
npm run test:unit
npm run generate
```

Общая команда:

```bash
npm run check
```

Команда `test:unit` должна запускать Vitest в non-watch режиме.

Также предусмотрено:

```bash
npm run test:e2e
```

E2E на текущем этапе может быть отложен до появления полноценного CI/CD.

## Что тестировать в первую очередь

Обязательно покрывать:

* demo-компоненты;
* компоненты с нетривиальной логикой.

В особенности обращать внимание на:

* валидацию;
* переключение состояний;
* ошибки внешнего API;
* трансформацию данных;
* поведение route-level логики, если она перестаёт быть тривиальной.

## Правила работы

* Если изменение меняет поведение, обнови спецификацию до реализации.
* Если логика нетривиальна, не оставляй её без unit-теста.
* Если проверка не запускалась, это нужно явно сообщить пользователю.
* Если часть проверок пока невозможна из-за состояния проекта, это тоже нужно явно фиксировать.

## Definition of Done

Изменение считается завершённым, если:

* спецификации обновлены;
* код реализован;
* typecheck проходит;
* lint проходит;
* unit-тесты проходят;
* статическая сборка выполняется;
* README обновлён при необходимости.
